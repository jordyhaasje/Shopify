# Manual PR Checklist

GitHub Actions is not active for this phase. Use this checklist before manual review or merge.

- Confirm the PR is scoped to the stated task.
- Confirm no `.github/workflows` files are added.
- Confirm documentation-only validation PRs do not change core write helpers, MCP execute implementations, Shopify client mutations, package publishing, or automated live-test behavior.
- Confirm no secrets, tokens, real customer data, or real order data appear in code, docs, tests, or logs.
- Run local validation:

```bash
pnpm run lint
pnpm run typecheck
pnpm test
pnpm run build
pnpm run smoke:local
```

- For package metadata changes, run `pnpm run pack:check` and confirm it is dry-run only, creates no tarballs, and includes only intended package files.
- For release or publishing docs, confirm package publishing remains manual, explicitly approved, and gated by [release-runbook.md](release-runbook.md).
- Review changed docs for product-boundary drift.
- For user-facing install/use docs, confirm [user-quickstart.md](user-quickstart.md) remains the compact first-run entry point and deeper docs link to it instead of duplicating the full flow.
- Review read-tool audit behavior: successful read, not found, and multiple matches use `success`; missing input uses `blocked`; Shopify/API/invalid response uses `failed`.
- For `inventory.lookup` changes, confirm it remains read-only, accepts exactly one explicit inventory item ID, variant ID, or SKU, returns only compact inventory item/variant/location/quantity summaries, does not write, does not run as hidden execute-time discovery, and does not return raw Shopify nodes or product/location dumps.
- For `inventory.locationLookup` changes, confirm it remains read-only, accepts exactly one explicit location ID, name, or query, returns only compact location summaries, does not write, does not run as hidden execute-time discovery, and does not return raw Shopify nodes, address dumps, or product/location dumps.
- Review preview-tool audit behavior: successful previews use `success`; missing input and validation errors use `blocked`.
- Confirm preview output summarizes large payloads, redacts secret-looking values, and does not autonomously fetch products or call Shopify write APIs.
- If preview execute helpers change, confirm they are review-only, appear only for implemented write previews, contain safe bounded reviewed payloads, require explicit `confirmed: true`, and do not make placeholder execute tools look like real writes.
- For `inventory.transfer.preview` changes, confirm it remains preview-only until execute is called, returns `executeRequest` only for `inventory.transfer.execute`, accepts only explicit inventory item ID, distinct source/destination location IDs, positive quantity, and reason, and performs no Shopify fetch/write.
- For `inventory.transfer.markReady.preview` changes, confirm it remains preview-only until execute is called, returns `executeRequest` only for `inventory.transfer.markReady.execute`, accepts only an explicit inventory transfer ID, and performs no Shopify fetch/write.
- For `inventory.transfer.cancel.preview` changes, confirm it remains preview-only until execute is called, returns `executeRequest` only for `inventory.transfer.cancel.execute`, accepts only an explicit inventory transfer ID, and performs no Shopify fetch/write.
- For `inventory.transfer.ship.preview` changes, confirm it remains preview-only until execute is called, returns `executeRequest` only for `inventory.transfer.ship.execute`, accepts only an explicit inventory transfer ID, explicit inventory item ID, and positive quantity, and performs no Shopify fetch/write.
- For `inventory.transfer.receive.preview` changes, confirm it remains preview-only until execute is called, returns `executeRequest` only for `inventory.transfer.receive.execute`, accepts only an explicit inventory shipment ID, explicit shipment line item ID, positive quantity, and receive reason `ACCEPTED` or `REJECTED`, and performs no Shopify fetch/write.
- Review local preview-store behavior when changed: stored previews are safe/summarized, hash output is deterministic, TTL expiry fails closed, and no raw reviewed payloads are returned.
- Review persistent audit behavior when changed: local JSONL entries are append-only, compact, and contain no secrets, raw Shopify nodes, raw reviewed payloads, customer/order dumps, or package-publish side effects.
- For optional preview read enrichment, confirm it is explicit opt-in, uses only explicit IDs/handles, returns minimal summaries, treats read failures as warnings, and performs no mutations.
- Review execute tools for read-only, preview binding, confirmation, and audit behavior.
- Confirm execute placeholders require preview ID plus reviewed payload/context; missing, expired, invalid, or mismatched binding uses `blocked`, and valid placeholders use `not_implemented`, never `success`.
- For `page.create.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `page.create.preview` binding, locally preflights `write_content` or `write_online_store_pages`, calls Shopify only after binding and scope preflight succeed, verifies by created page ID only after create success, returns safe Shopify `userErrors`, audits success only after a successful page create, and never returns raw reviewed payloads or raw Shopify nodes.
- For `product.create.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `product.create.preview` binding, locally preflights `write_products`, calls Shopify only after binding and scope preflight succeed, performs only the product create mutation, supports only title, description/body HTML summary, vendor, product type, status, and tags, returns a safe created product summary, audits success only after a successful product create, and never returns raw reviewed payloads or raw Shopify nodes.
- For `product.update.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `product.update.preview` binding, locally preflights `write_products`, calls Shopify only after binding and scope preflight succeed, performs only one supported mutation shape per preview (`productUpdate` for basic product fields, `productVariantsBulkUpdate` for explicit variant price updates, `productVariantsBulkCreate` for explicit variant creation, `productOptionsCreate` with `LEAVE_AS_IS` for explicit option creation, `productOptionsDelete` with `NON_DESTRUCTIVE` for explicit option delete, `productOptionsReorder` for explicit option reorder, or `productOptionUpdate` with `LEAVE_AS_IS` for explicit option rename, option value rename, option value add, or option value delete), blocks mixed update-shape previews before fetch, supports only title, description/descriptionHtml, vendor, product type, status, tags, explicit variant IDs with prices, explicit variant option values with optional price/SKU, explicit option names and values, explicit option IDs to delete, explicit option IDs or names in desired order, explicit option ID plus new name, explicit option ID plus option value ID and new value name, explicit option ID plus new value names, or explicit option ID plus option value IDs to delete from the stored preview, fails closed for handle-only previews without a safe product ID, returns a safe updated product, variant-price, created-variant, created-option, deleted-option, reordered-option, renamed-option, renamed-option-value, added-option-values, or deleted-option-values summary, audits success only after a successful Shopify write, and never returns raw reviewed payloads or raw Shopify nodes.
- For `collection.create.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `collection.create.preview` binding, locally preflights `write_products`, calls Shopify only after binding and scope preflight succeed, performs only the collection create mutation, supports only title, optional handle, and explicit product IDs from the stored preview, fails closed for rule-based or smart collection previews, returns a safe created collection summary, audits success only after a successful collection create, and never returns raw reviewed payloads, raw product dumps, or raw Shopify nodes.
- For `inventory.setQuantity.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `inventory.setQuantity.preview` binding, locally preflights `write_inventory`, calls Shopify only after binding and scope preflight succeed, performs only one `inventorySetQuantities` mutation for one explicit inventory item ID and location ID, supports only quantity name `available`, requires compare quantity unless explicitly ignored in preview, derives idempotency from stored preview binding, returns a safe inventory-set summary, audits success only after a successful Shopify write, and never returns raw reviewed payloads, raw Shopify nodes, product dumps, location dumps, or unrelated execute input.
- For `inventory.adjustQuantity.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `inventory.adjustQuantity.preview` binding, locally preflights `write_inventory`, calls Shopify only after binding and scope preflight succeed, performs only one `inventoryAdjustQuantities` mutation for one explicit inventory item ID and location ID, supports only quantity name `available` with one non-zero delta, derives idempotency from stored preview binding, returns a safe inventory-adjustment summary, audits success only after a successful Shopify write, and never returns raw reviewed payloads, raw Shopify nodes, product dumps, location dumps, or unrelated execute input.
- For `inventory.moveQuantity.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `inventory.moveQuantity.preview` binding, locally preflights `write_inventory`, calls Shopify only after binding and scope preflight succeed, performs only one `inventoryMoveQuantities` mutation for one explicit inventory item ID and location ID, moves only a positive quantity between supported source and destination quantity names at the same explicit location, derives idempotency from stored preview binding, returns a safe inventory-move summary, audits success only after a successful Shopify write, and never returns raw reviewed payloads, raw Shopify nodes, product dumps, location dumps, or unrelated execute input.
- For `inventory.transfer.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `inventory.transfer.preview` binding, locally preflights both `write_inventory_transfers` and `read_inventory_transfers`, calls Shopify only after binding and scope preflight succeed, performs only one `inventoryTransferCreate` mutation for one explicit inventory item ID, source location ID, destination location ID, and quantity, creates only a draft transfer, derives idempotency from stored preview binding, returns a safe inventory-transfer summary, audits success only after a successful Shopify write, and never returns raw reviewed payloads, raw Shopify nodes, product dumps, location dumps, or unrelated execute input.
- For `inventory.transfer.markReady.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `inventory.transfer.markReady.preview` binding, locally preflights both `write_inventory_transfers` and `read_inventory_transfers`, calls Shopify only after binding and scope preflight succeed, performs only one `inventoryTransferMarkAsReadyToShip` mutation for one explicit inventory transfer ID, does not create/edit/ship/receive/cancel transfers, returns a safe inventory-transfer summary, audits success only after a successful Shopify write, and never returns raw reviewed payloads, raw Shopify nodes, product dumps, location dumps, transfer dumps, or unrelated execute input.
- For `inventory.transfer.cancel.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `inventory.transfer.cancel.preview` binding, locally preflights both `write_inventory_transfers` and `read_inventory_transfers`, calls Shopify only after binding and scope preflight succeed, performs only one `inventoryTransferCancel` mutation for one explicit inventory transfer ID, does not create/edit/mark-ready/ship/receive transfers, returns a safe inventory-transfer summary, audits success only after a successful Shopify write, and never returns raw reviewed payloads, raw Shopify nodes, product dumps, location dumps, transfer dumps, or unrelated execute input.
- For `inventory.transfer.ship.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `inventory.transfer.ship.preview` binding, locally preflights both `write_inventory_shipments` and `read_inventory_shipments`, calls Shopify only after binding and scope preflight succeed, performs only one `inventoryShipmentCreateInTransit` mutation for one explicit inventory transfer ID, inventory item ID, and quantity, does not create/edit/mark-ready/cancel/receive transfers, returns a safe inventory-shipment summary, audits success only after a successful Shopify write, and never returns raw reviewed payloads, raw Shopify nodes, product dumps, location dumps, transfer/shipment dumps, or unrelated execute input.
- For `inventory.transfer.receive.execute` changes, confirm it is blocked by read-only mode, requires a valid stored `inventory.transfer.receive.preview` binding, locally preflights both `write_inventory_shipments_received_items` and `read_inventory_shipments`, calls Shopify only after binding and scope preflight succeed, performs only one `inventoryShipmentReceive` mutation for one explicit inventory shipment ID, shipment line item ID, quantity, and receive reason, does not create/edit/mark-ready/cancel/ship transfers, returns a safe inventory-shipment summary, audits success only after a successful Shopify write, and never returns raw reviewed payloads, raw Shopify nodes, product dumps, location dumps, transfer/shipment dumps, or unrelated execute input.
- Review capability or auth changes for token redaction and mocked tests only.
- Review setup wizard changes for read-only defaults, OAuth auth read-only default scopes, explicit handling of requested `write_` scopes, local-only config storage, safe capability checks, host snippets without raw secrets, local GitHub build MCP commands while npm publishing is inactive, and no Shopify write-scope requirement for read/preview setup.
- Review `setup-check` changes for local-only behavior, `fetchCalls: 0`, no Shopify writes or live fetches, no raw token/config output, safe host snippets, read-only onboarding checks, and clear remediation steps for missing config/auth/build state.
- Confirm `setup --auth oauth` remains guidance/snippet-only and does not imply token exchange or overwrite a working OAuth token config; the real local OAuth browser flow belongs to `auth`.
- Confirm write-mode warnings name `page.create.execute`, `product.create.execute`, `product.update.execute`, `collection.create.execute`, `inventory.setQuantity.execute`, `inventory.adjustQuantity.execute`, `inventory.moveQuantity.execute`, `inventory.transfer.execute`, `inventory.transfer.markReady.execute`, `inventory.transfer.cancel.execute`, `inventory.transfer.ship.execute`, and `inventory.transfer.receive.execute` as the only implemented execute tools, with all other execute tools remaining fail-closed placeholders.
- Review smoke validation changes for local/mocked defaults, no default fetches, no writes or mutations, preview-store coverage, and execute placeholders auditing `blocked` or `not_implemented` but never `success`.
- Review dev-store runbook changes for manual-only language, development-store-only writes, no production customer store guidance, persistent preview-store caveats, read-only defaults, explicit write scopes, and `fetchCalls: 0` local smoke expectations.
- For manual dev-store validation evidence, confirm the PR or issue records only safe pass/fail/skipped summaries, keeps `smoke:local` at `fetchCalls: 0`, avoids raw Shopify responses and reviewed payloads, and does not include secrets, customer data, order data, or production product data.
- For live dev-store E2E runs, confirm `e2e-preflight` passed against the exact expected store/config path before any write step, with `no_fetch: true` in the preflight result.
- Merge manually only after local validation and reviewer approval.
