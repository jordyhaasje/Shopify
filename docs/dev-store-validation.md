# Dev Store Validation

Use this checklist before testing Shopify Store Agent against a development store. Do not use a customer production store for first validation. For the full manual MVP flow, including setup, MCP host connection, previews, `page.create.execute`, `product.create.execute`, `product.update.execute`, `collection.create.execute`, negative tests, and audit checks, see [dev-store-e2e-runbook.md](dev-store-e2e-runbook.md).

## Local Smoke

Run the local smoke validation first:

```bash
pnpm run smoke:local
```

The default smoke path is local/mocked only. It should not make Shopify network calls, should not perform writes, and should not run GraphQL mutations.

Expected local smoke output includes:

```text
fetchCalls: 0
```

Optional live mode is limited to the minimal capability check:

```bash
pnpm --filter shopify-store-agent run smoke -- --live --admin-token "$SHOPIFY_ADMIN_TOKEN"
```

## Setup

- Use a development store or disposable test store.
- Use the current GitHub clone + `pnpm install` + `pnpm run build` route while npm publishing is inactive.
- Prefer local OAuth with the `auth` command when Shopify app client credentials are available; use manual Admin API token setup only as a fallback.
- Treat `setup --auth oauth` as guidance/snippet generation only. It does not run the browser flow or exchange a token.
- Keep setup read-only by default.
- Use read-only Admin API scopes for read and preview validation.
- Do not request write scopes for read, preview, setup, or smoke validation. For limited write tests, use only a development store and the minimum write scope needed for that execute path: `write_content` or `write_online_store_pages` for `page.create.execute`, `write_products` for `product.create.execute`, basic-field, explicit-variant-price, explicit-variant-create, explicit-option-create, explicit-option-delete, explicit-option-reorder, explicit-option-rename, explicit-option-value-rename, explicit-option-value-add, or explicit-option-value-delete `product.update.execute`, or custom explicit-product `collection.create.execute`, `write_inventory` for `inventory.setQuantity.execute`, `inventory.adjustQuantity.execute`, or `inventory.moveQuantity.execute`, both `write_inventory_transfers` and `read_inventory_transfers` for `inventory.transfer.execute`, `inventory.transfer.markReady.execute`, or `inventory.transfer.cancel.execute`, add `read_inventory` for `inventory.transfer.addItems.execute`, both `write_inventory_shipments` and `read_inventory_shipments` for `inventory.transfer.ship.execute`, and both `write_inventory_shipments_received_items` and `read_inventory_shipments` for `inventory.transfer.receive.execute`. Read-only mode must be explicitly disabled for each deliberate write test.
- Run setup and review the generated MCP host snippet.
- Confirm snippets point to a local config path, use the local `node /absolute/path/to/Shopify/packages/mcp/dist/server.js` command in the current GitHub-only phase, and do not print raw tokens.
- Remember that npm/npx snippets are a future package-published route.
- Before live write E2E, run `pnpm --filter shopify-store-agent run e2e-preflight -- --store your-store.myshopify.com --config /absolute/path/to/config.json --required-scopes "read_products,read_content,read_online_store_pages,read_inventory,read_locations,write_products,write_content,write_inventory,write_inventory_transfers,read_inventory_transfers,write_inventory_shipments,read_inventory_shipments,write_inventory_shipments_received_items" --require-write-enabled` and confirm the local-only preflight passes with `no_fetch: true`.

## Manual Validation

When documenting a manual run, use the evidence format in [dev-store-e2e-runbook.md](dev-store-e2e-runbook.md). Keep the record in the relevant PR or issue unless the result becomes durable product documentation. Record pass/fail/skipped status and safe summaries only; do not paste tokens, raw Shopify responses, raw reviewed payloads, config files, customer data, order data, or production product data.

- Run `shopify.capabilities.check` in local mode first.
- Test one read tool with explicit user-provided input. `inventory.lookup` may be used read-only with an explicit inventory item ID, variant ID, or SKU to collect test inventory IDs before a reviewed inventory preview. `inventory.locationLookup` may be used read-only with an explicit location ID, name, or query to collect location IDs for review.
- Test one catalog/content preview tool.
- Confirm the preview store receives a record.
- Test an execute placeholder with invalid binding and confirm `blocked`.
- Test an execute placeholder with valid stored binding and confirm `not_implemented`.
- Confirm execute placeholders never audit `success`.
- If testing `page.create.execute`, create a `page.create.preview`, review the stored binding payload, then run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing `write_content` or `write_online_store_pages`, and read-only mode disabled.
- If testing `product.create.execute`, create a `product.create.preview`, review the stored binding payload, then run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing `write_products`, and read-only mode disabled. Confirm the mutation is limited to product create with title, description/body HTML summary, vendor, product type, status, and tags only.
- If testing `product.update.execute`, create a `product.update.preview` that includes a safe product ID, review the stored binding payload, then run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing `write_products`, and read-only mode disabled. Confirm the mutation is limited to one supported shape: `productUpdate` with title, description/descriptionHtml, vendor, product type, status, and tags; `productVariantsBulkUpdate` with explicit variant IDs and prices; `productVariantsBulkCreate` with explicit option values and optional price/SKU; `productOptionsCreate` with explicit option names and values using `LEAVE_AS_IS`; `productOptionsDelete` with explicit option IDs using `NON_DESTRUCTIVE`; `productOptionsReorder` with explicit option IDs or names in the desired order; `productOptionUpdate` with explicit option ID and new name using `LEAVE_AS_IS`; `productOptionUpdate` with explicit option ID, option value ID, and new value name using `LEAVE_AS_IS`; `productOptionUpdate` with explicit option ID and new value names using `LEAVE_AS_IS`; or `productOptionUpdate` with explicit option ID and option value IDs to delete using `LEAVE_AS_IS`. Confirm mixed update-shape previews fail closed before fetch.
- If testing `collection.create.execute`, create a `collection.create.preview` with explicit product IDs, review the stored binding payload, then run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing `write_products`, and read-only mode disabled. Confirm the mutation is limited to custom collection create with title, optional handle, and explicit product IDs only.
- If testing `inventory.setQuantity.execute`, create an `inventory.setQuantity.preview` with explicit inventory item ID, explicit location ID, quantity, reason, and compare quantity or explicit ignore compare quantity. Run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing `write_inventory`, and read-only mode disabled. Confirm the mutation is limited to one `inventorySetQuantities` call for quantity name `available`.
- If testing `inventory.adjustQuantity.execute`, create an `inventory.adjustQuantity.preview` with explicit inventory item ID, explicit location ID, non-zero delta, and reason. Run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing `write_inventory`, and read-only mode disabled. Confirm the mutation is limited to one `inventoryAdjustQuantities` call for quantity name `available`.
- If testing `inventory.moveQuantity.execute`, create an `inventory.moveQuantity.preview` with explicit inventory item ID, explicit location ID, positive quantity, source quantity name, destination quantity name, and reason. Run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing `write_inventory`, and read-only mode disabled. Confirm the mutation is limited to one `inventoryMoveQuantities` call between states at the same explicit location.
- If testing `inventory.transfer.execute`, create an `inventory.transfer.preview` with explicit inventory item ID, distinct source/destination location IDs, positive quantity, and reason. Run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing both `write_inventory_transfers` and `read_inventory_transfers`, and read-only mode disabled. Confirm the mutation is limited to one `inventoryTransferCreate` call that creates a draft transfer.
- If testing `inventory.transfer.markReady.execute`, create an `inventory.transfer.markReady.preview` with explicit inventory transfer ID. Run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing both `write_inventory_transfers` and `read_inventory_transfers`, and read-only mode disabled. Confirm the mutation is limited to one `inventoryTransferMarkAsReadyToShip` call.
- If testing `inventory.transfer.cancel.execute`, create an `inventory.transfer.cancel.preview` with explicit inventory transfer ID. Run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing both `write_inventory_transfers` and `read_inventory_transfers`, and read-only mode disabled. Confirm the mutation is limited to one `inventoryTransferCancel` call.
- If testing `inventory.transfer.ship.execute`, create an `inventory.transfer.ship.preview` with explicit inventory transfer ID, inventory item ID, and positive quantity. Run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing both `write_inventory_shipments` and `read_inventory_shipments`, and read-only mode disabled. Confirm the mutation is limited to one `inventoryShipmentCreateInTransit` call.
- If testing `inventory.transfer.receive.execute`, create an `inventory.transfer.receive.preview` with explicit inventory shipment ID, shipment line item ID, positive quantity, and receive reason `ACCEPTED` or `REJECTED`. Run execute only with `confirmed: true`, matching target/tool/hash values, local granted scopes showing both `write_inventory_shipments_received_items` and `read_inventory_shipments`, and read-only mode disabled. Confirm the mutation is limited to one `inventoryShipmentReceive` call.
- Confirm missing or unknown local write scopes block before any Shopify fetch.
- Confirm successful page create performs only the page create mutation followed by verification of the created page ID.
- Confirm successful product create performs only the product create mutation and does not create variants, inventory, media, collections, metafields, publications, translations, updates, deletes, or bulk operations.
- Confirm successful product update performs only the expected narrow mutation: `productUpdate` for basic fields, `productVariantsBulkUpdate` for explicit variant price updates, `productVariantsBulkCreate` for explicit variant creation, `productOptionsCreate` for explicit option creation with `LEAVE_AS_IS`, `productOptionsDelete` for explicit option delete with `NON_DESTRUCTIVE`, `productOptionsReorder` for explicit option reorder, `productOptionUpdate` for explicit option rename with `LEAVE_AS_IS`, `productOptionUpdate` for explicit option value rename with `LEAVE_AS_IS`, `productOptionUpdate` for explicit option value add with `LEAVE_AS_IS`, or `productOptionUpdate` for explicit option value delete with `LEAVE_AS_IS`. Confirm it does not update inventory, media, collections, metafields, SEO, publications, translations, deletes, or bulk operations. Confirm handle-only previews fail closed unless a safe product ID is present in the stored preview.
- Confirm successful collection create performs only the collection create mutation and does not create smart/rule-based collections, publish the collection, update SEO, update metafields, upload media/images, update navigation, discover products, or run bulk operations. Confirm rule-based collection previews fail closed at execute.
- Confirm successful inventory set quantity performs only `inventorySetQuantities`, uses explicit inventory item and location IDs from the stored preview, does not run lookup/discovery during execute, does not perform bulk inventory, state move, inter-location transfer, or product field updates, and does not return raw location/product dumps.
- Confirm successful inventory quantity adjustment performs only `inventoryAdjustQuantities`, uses explicit inventory item and location IDs plus delta from the stored preview, does not run lookup/discovery during execute, does not perform bulk inventory, state move, inter-location transfer, or product field updates, and does not return raw location/product dumps.
- Confirm successful inventory quantity move performs only `inventoryMoveQuantities`, uses explicit inventory item and location IDs plus source/destination quantity names from the stored preview, does not run lookup/discovery during execute, does not perform bulk inventory, inter-location transfer, or product field updates, and does not return raw location/product dumps.
- Confirm successful inventory transfer performs only `inventoryTransferCreate`, uses explicit inventory item, source location, destination location, and quantity from the stored preview, creates only a draft transfer, does not run lookup/discovery during execute, does not perform bulk inventory, transfer lifecycle actions, location management, or product field updates, and does not return raw location/product dumps.
- Confirm successful inventory transfer add-items performs only `inventoryTransferSetItems`, uses the explicit inventory transfer ID, inventory item ID, and quantity from the stored preview, does not run lookup/discovery during execute, does not create/remove/mark-ready/ship/receive/cancel transfers, does not perform bulk inventory, location management, or product field updates, and does not return raw location/product/transfer dumps.
- Confirm successful inventory transfer mark-ready performs only `inventoryTransferMarkAsReadyToShip`, uses the explicit inventory transfer ID from the stored preview, does not run lookup/discovery during execute, does not create/edit/ship/receive/cancel transfers, does not perform bulk inventory, location management, or product field updates, and does not return raw location/product/transfer dumps.
- Confirm successful inventory transfer cancel performs only `inventoryTransferCancel`, uses the explicit inventory transfer ID from the stored preview, does not run lookup/discovery during execute, does not create/edit/mark-ready/ship/receive transfers, does not perform bulk inventory, location management, or product field updates, and does not return raw location/product/transfer dumps.
- Confirm successful inventory transfer ship performs only `inventoryShipmentCreateInTransit`, uses the explicit inventory transfer ID, inventory item ID, and quantity from the stored preview, does not run lookup/discovery during execute, does not create/edit/mark-ready/cancel/receive transfers, does not perform bulk inventory, location management, or product field updates, and does not return raw location/product/transfer/shipment dumps.
- Confirm successful inventory transfer receive performs only `inventoryShipmentReceive`, uses the explicit inventory shipment ID, shipment line item ID, quantity, and reason from the stored preview, does not run lookup/discovery during execute, does not create/edit/mark-ready/cancel/ship transfers, does not perform bulk inventory, location management, or product field updates, and does not return raw location/product/transfer/shipment dumps.
- Confirm all execute tools except `page.create.execute`, `product.create.execute`, `product.update.execute`, `collection.create.execute`, `inventory.setQuantity.execute`, `inventory.adjustQuantity.execute`, `inventory.moveQuantity.execute`, `inventory.transfer.execute`, `inventory.transfer.addItems.execute`, `inventory.transfer.markReady.execute`, `inventory.transfer.cancel.execute`, `inventory.transfer.ship.execute`, and `inventory.transfer.receive.execute` still remain placeholders.
- Review the audit log for safe targets and no raw payload dumps.

## Safety

- Confirm local smoke, setup, read, preview, invalid execute binding, and placeholder execute checks perform no writes or mutations.
- Confirm any deliberate write test is limited to `page.create.execute`, `product.create.execute`, basic-field, explicit-variant-price, explicit-variant-create, explicit-option-create, explicit-option-delete, explicit-option-reorder, explicit-option-rename, explicit-option-value-rename, explicit-option-value-add, or explicit-option-value-delete `product.update.execute`, custom explicit-product `collection.create.execute`, explicit single-item `inventory.setQuantity.execute`, explicit single-item `inventory.adjustQuantity.execute`, explicit same-location state `inventory.moveQuantity.execute`, explicit single-item draft transfer `inventory.transfer.execute`, explicit transfer add-item quantity set `inventory.transfer.addItems.execute`, explicit transfer mark-ready `inventory.transfer.markReady.execute`, explicit transfer cancel `inventory.transfer.cancel.execute`, explicit transfer ship `inventory.transfer.ship.execute`, or explicit transfer receive `inventory.transfer.receive.execute` in a development store.
- Confirm no raw Admin API tokens, OAuth client secrets, Theme Access tokens, customer data, order data, or product dumps appear in logs.
- Keep read-only mode enabled except for explicit reviewed `page.create.execute`, `product.create.execute`, basic-field, explicit-variant-price, explicit-variant-create, explicit-option-create, explicit-option-delete, explicit-option-reorder, explicit-option-rename, explicit-option-value-rename, explicit-option-value-add, or explicit-option-value-delete `product.update.execute`, custom explicit-product `collection.create.execute`, explicit single-item `inventory.setQuantity.execute`, explicit single-item `inventory.adjustQuantity.execute`, explicit same-location state `inventory.moveQuantity.execute`, explicit single-item draft transfer `inventory.transfer.execute`, explicit transfer add-item quantity set `inventory.transfer.addItems.execute`, explicit transfer mark-ready `inventory.transfer.markReady.execute`, explicit transfer cancel `inventory.transfer.cancel.execute`, explicit transfer ship `inventory.transfer.ship.execute`, or explicit transfer receive `inventory.transfer.receive.execute` development-store tests.
- Remember that the MCP default context persists safe preview records locally. Missing, corrupt, expired, or mismatched preview records still fail closed; create a new preview when in doubt.
